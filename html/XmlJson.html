Semejanzas y diferencias entre XML y JSON

¿Qué es XML?
El lenguaje de marcado extensible (XML) permite definir y almacenar datos de forma compartible. XML admite el intercambio de información entre sistemas de computación, como sitios web, bases de datos y aplicaciones de terceros.
  • XML significa lenguaje de marcado extensible
  • XML es un lenguaje de marcado muy parecido a HTML
  • XML fue diseñado para almacenar y transportar datos
  • XML fue diseñado para ser autodescriptivo
Los elementos XML son extensibles
Los elementos XML se pueden ampliar para llevar más información.
Mire el siguiente ejemplo XML:

Código

<note>
  <to>Tove</to>
  <from>Jani</from>
  <body>Don't forget me this weekend!</body>
</note>

Ejemplo

From: Jani

Don't forget me this weekend!

¿Qué es JSON?
JSON, que significa JavaScript Object Notation, es una formatación usada para estructurar datos en forma de texto y transmitirlos de un sistema a otro, como en aplicaciones cliente-servidor o en aplicaciones móviles.
Sintaxis JSON
La sintaxis de JSON se deriva de la sintaxis de notación de objetos de JavaScript:
  • Los datos están en pares de nombre/valor
  • Los datos están separados por comas
  • Las llaves sostienen objetos
  • Los corchetes contienen matrices
Valores JSON
En JSON , los valores deben ser uno de los siguientes tipos de datos:
  • una cuerda
  • un número
  • un objeto
  • una matriz
  • un booleano
  • nulo
Mire el siguiente ejemplo:

Código

const obj = {name: "John", age: 30, city: "New York"};
const myJSON = JSON.stringify(obj);

Diferencias entre XML y JSON
XML JSON
XML significa lenguaje de marcas extensible. XML es legible por los humanos pero no necesariamente por las máquinas. JSON es ligero y fácil de leer, pero no proporciona información sobre el esquema o el tipo. Es ideal para compartir datos entre varias aplicaciones.
XML es una forma más larga de representar datos estructurados. JSON es una forma ligera y legible para el ser humano de representar estructuras de datos.
XML sólo puede almacenarse como archivo de texto. JSON puede utilizarse con archivos JavaScript o de texto plano. Además, JSON utiliza menos memoria que el programa XML a la hora de procesar la información.
XML se utiliza para representar datos de forma legible por la máquina. El formato JSON se utiliza para almacenar y transmitir datos.
XML sigue siendo la opción preferida para transmitir datos estructurados a través de la web. JSON ha ido ganando popularidad como medio de almacenamiento para aplicaciones web debido a su simplicidad.
XML tiene algunos problemas de compatibilidad entre plataformas. No es compatible con muchos lenguajes de programación (excepto Actionscript). Puedes utilizar JSON en tu aplicación web o móvil sin preocuparte de los problemas de compatibilidad porque está ampliamente aceptado en la web y en las aplicaciones móviles.
